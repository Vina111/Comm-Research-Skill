# 规则目的

统一企业与活动风险分类、状态和优先级表达。

# 企业风险分类

- 企业治理与财务；
- 产品、技术、安全与质量；
- 数据、隐私、AI 与网络安全；
- 供应链、生产、物流与劳工；
- 法律、监管、合规与知识产权；
- 市场、客户、渠道与平台依赖；
- 合作伙伴与利益相关方；
- 声誉、媒体、ESG 与 greenwashing；
- 地缘政治、贸易、制裁与出口管制。

# 活动风险分类

- 主办方、所有权或合作伙伴声誉；
- 活动定位、规模或参会数据夸大；
- 参展商、赞助商或嘉宾争议；
- 奖项公信力、付费属性或利益冲突；
- 官方活动与场外活动混淆；
- 议程、发言权、媒体邀请权和素材控制权；
- 媒体到场数据或报道数据误读；
- 安全、场地、物流、签证和突发事件；
- 政策、抗议、地缘政治与敏感议题。

# 风险状态

- 已发生；
- 正在发展；
- 潜在暴露；
- 行业/活动通用风险。

# 优先级规则

综合 Evidence、Likelihood、Potential Impact 和 Trigger 判断。不得仅因风险听起来严重就标为高风险。

# 禁止项

- 不把行业或活动通用风险写成研究对象已发生事件。
- 不自动生成危机应对方案或回应口径。
