# 规则目的

统一展会、会议、峰会、论坛及大型行业活动的研究定义、分类和比较标准。

# 活动类型

先确认研究对象属于何种类型：

- Trade Show / Expo：以展览、参展商和商业洽谈为主；
- Conference / Congress：以会议议程、演讲和专业交流为主；
- Summit / Forum：以高层决策者、行业议题和政策讨论为主；
- Festival / Showcase：以体验、内容和公众参与为主；
- Hybrid Event：同时包含展览、论坛、奖项、发布和社交活动。

活动类型影响参与者分类和数据解释，不得套用同一规模标准。

# 主体角色分类

必须区分：

- Organizer / Owner；
- Co-organizer / Institutional Partner；
- Exhibitor；
- Sponsor；
- Speaker / Keynote；
- Award Entrant / Winner；
- Media Partner；
- Attending Media / Journalist；
- Reporting Media / Author；
- Visitor / Delegate / Buyer；
- Side-event Organizer。

同一实体可以有多个角色，但每个角色必须分别记录。

# 规模数据规则

以下指标不得互换：

- 注册人数；
- 实际到场人数；
- 独立访客人数；
- Delegate 数；
- 参展商数量；
- 展位数量；
- 参展品牌数量；
- 演讲者数量；
- 国家/地区数量；
- 媒体注册数量。

跨届比较前必须核查统计定义、活动天数、线上/线下模式和主办方口径是否一致。

# 参展商组成规则

参展商分析至少考虑：

- 行业或产品类别；
- 企业总部或主要来源地；
- 企业类型与发展阶段；
- 展区、主题馆或会议轨道；
- 新参展与连续参展；
- 赞助层级；
- 是否同时演讲、获奖或举办发布。

不应仅罗列知名公司，应分析整体构成、集中度和变化。

# 发布与活动边界

“在活动上发布”必须有时间和关联证据。统一分类为：

1. Official Program Launch：出现在官方议程或新闻中心；
2. Onsite Brand Press Conference：品牌在场馆内自办发布会；
3. Side / Co-located Launch：同期活动或联合活动发布；
4. Offsite Brand Event：同城、同期但不属于官方活动；
5. Pre-show Announcement：展前预热或提前公告；
6. Demo / Showcase Only：仅展示，无正式发布证据；
7. Unable to Verify Launch：无法确认。

# 奖项分类

必须区分：

- Official Organizer Award：主办方官方奖项；
- Official Partner Award：主办方认可的合作奖项；
- Media / Association Award：媒体或协会评选；
- Independent Third-party Award：独立第三方奖项；
- Commercial / Paid-entry Recognition：可能存在付费申报或商业合作属性的荣誉。

每个奖项需核实：主办方、官方关系、资格、评审、类别、时间、申报方式和获奖名单。不得仅凭名称推定权威性。

# 内容角度分类

活动报道可按以下主角度编码；如一篇包含多个角度，应确定主角度并保留次级标签：

- Event Overview / What to Expect；
- Industry Trend / Market Outlook；
- Product Launch / New Technology；
- Corporate Strategy / Executive Keynote；
- Partnership / Investment / Deal；
- Policy / Regulation / Standards；
- Award / Recognition；
- Experience / Demo / Design；
- Roundup / Best of Show；
- Sustainability / ESG；
- Controversy / Risk / Criticism；
- Logistics / Attendance / Event Operations。

角度占比应以去重后的独立内容为基础。

# 关注度排名规则

“最受关注”必须在 Research Scope Confirmation 中定义口径，不能仅凭主观判断。可使用：

- 去重后的独立报道数；
- 标题提及数；
- 估算触达；
- 社交互动；
- Share of Voice；
- 高权威媒体覆盖数；
- 跨市场覆盖广度。

推荐至少使用两个指标并说明权重。关注度与正负面情绪必须分开，不得把争议造成的高声量写成正面成功。

# 媒体与到场规则

- Meltwater 证明的是公开内容，不等于到场。
- 到场媒体需有认证、注册、签到、采访安排、组织方名单或现场证据。
- Media Partner 身份不等于实际派员到场。
- 转载同一稿件的媒体不代表多个独立报道角度。

# 活动研究来源优先级

1. 官方历史档案、议程、参展商目录、Press Room、奖项规则；
2. 主办方审计或会后报告；
3. 场馆、协会、政府和合作机构资料；
4. 用户提供的注册、认证、Meltwater 和内部材料；
5. 品牌新闻中心和正式发布；
6. 独立媒体报道；
7. 社交内容与活动平台页面；
8. 二手汇总与搜索摘要。

主办方公布的规模和“成功”评价应标记为官方自述；有第三方审计时优先使用审计数据。
