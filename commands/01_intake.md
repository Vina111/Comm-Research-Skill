# 目标

在用户明确调用 Strategy Research Skill 后，通过最少必要问题确认研究需求；本命令不开展正式研究。

# 前置门槛

- 用户必须已明确要求使用 Strategy Research Skill；
- 如未明确调用，停止本命令并按普通 Research 请求处理；
- 仅读取 `rules/00_activation-and-context-efficiency.md` 与 `rules/01_intake-gate.md`。

# 输入

从用户当前消息中提取已经明确的信息，不重复询问。

# 执行步骤

1. 判断研究对象：企业 / 品牌 / 产品 / 活动 / 企业与活动组合。
2. 提取用户已说明的用途、核心问题、市场、时间和资料。
3. 只针对缺失且会改变研究路径的信息提问。
4. 第一轮问题原则上控制在 5-8 个主题内，可合并提问。
5. 用户回答后，检查是否仍有阻塞项；只有必要时发起第二轮补充问题。
6. 信息足够后，输出 `Research Intake Status`，进入 `/define-scope`；不得直接研究。

# 优先确认问题

根据任务选择，不机械全部询问：

1. **研究对象与用途**：具体公司、品牌、产品、展会或活动；研究将用于什么。
2. **核心问题**：最希望研究回答哪些问题，哪些问题优先。
3. **业务/活动属性**：B2B、B2C、B2B2C 或待判断；活动具体届次和比较年份。
4. **市场维度**：已有指定市场，还是需要先做市场初筛；是否涉及多个国家、地区或语言。
5. **时间范围**：完整历史、近几年、特定发布期或活动周期。
6. **模块选择**：企业核心、生态供应链、市场竞品、受众、媒体历史、风险、活动背景、参与者、议程奖项等。
7. **资料与数据**：客户材料、历史报告、官方名单、Meltwater 数据、Query 和筛选条件。
8. **交付要求与限制**：完整 Dossier、简报、PPT 研究部分、截止时间、敏感内容。
