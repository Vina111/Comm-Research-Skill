# 规则目的

定义 Strategy Research Skill 从显式调用到正式研究启动前必须满足的条件。

# Activation Gate

只有用户明确要求使用 Strategy Research Skill 时，才可进入 `/intake`。

普通单点 Research 请求不满足该门槛，按普通任务处理。

# Intake Gate

满足以下条件才可进入 `/define-scope`：

- 已明确研究对象与对象类型；
- 已明确研究用途和至少一个核心问题；
- 已明确地区/市场范围，或明确授权先做市场初筛；
- 已明确主要时间范围；
- 已知现有材料与关键限制；
- 已初步确定需要和不需要的研究模块。

# 企业任务附加条件

- 已确认或允许研究判断 B2B / B2C / B2B2C 属性。

# 活动任务附加条件

- 已确认活动名称和活动类型；
- 已确认目标届次/年份及是否比较往届；
- 已区分举办地、参与者来源范围、媒体市场和报道语言；
- 如需分析媒体、记者、报道角度或高关注品牌，已确认将提供 Meltwater 数据；
- 如需判断实际到场媒体/记者，已确认是否有认证、注册或到场名单。

# Scope Confirmation Gate

完成 `/define-scope` 后，必须等待用户确认。

只有用户明确回复“确认、开始、可以”或其他明确同意后，才可执行正式研究命令。

用户提出调整时，应更新 Scope 后再次确认。

# 提问规则

- 不重复询问用户已提供的信息；
- 只询问会改变研究路径、边界或数据口径的问题；
- 第一轮尽量将相关问题合并，避免逐条往返；
- 仅在仍有阻塞项时进行第二轮追问；
- 不要求用户回答可由公开研究解决的问题；
- 用户无法确认业务属性时，标记为“待研究判断”。

# 禁止项

不得默认美国、全球、英语、12 个月、消费者销售目标、媒体曝光目标、任一活动届次或完整模块集合。
