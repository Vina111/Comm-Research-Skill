# 规则目的

定义企业与活动 Meltwater 分析的数据门槛、可比性和禁止行为。

# 必须由用户提供或确认

- 分析目标；
- Meltwater 导出数据；
- 起止时间或活动届次窗口；
- 国家/地区；
- 语言；
- Search / Boolean Query；
- 品牌、活动名称、别名、缩写和 Hashtag；
- 排除词；
- 数据源；
- 去重规则；
- Sentiment 判定方式；
- 对比组与事件节点（如适用）。

# 活动关注度分析附加条件

需进一步确认：

- 是否仅统计参展商，或包含主办方、赞助商、演讲嘉宾和非参展品牌；
- 公司、品牌、子品牌和产品的实体合并规则；
- “最受关注”的排名指标；
- 是否排除新闻稿转载；
- 是否分别呈现正面、负面和中性关注；
- 不同届次是否使用相同 Query 和数据源。

# 到场媒体判定

- Meltwater 只能证明媒体或作者发布过相关内容，不能单独证明其实际到场。
- “到场媒体/记者”必须有媒体认证、注册、签到、组织方名单、采访日程或其他可靠证据。
- 只有报道证据时，统一使用“报道媒体/作者”或“覆盖该活动的媒体/作者”。

# 判定规则

- 任一关键口径缺失时，先索取，不开始分析。
- 不自行设置语言、地区、时间、关键词、竞品或活动别名。
- 不使用公开搜索结果伪装或替代 Meltwater 数据。
- 系统情绪与人工校正必须区分。
- 新闻稿转载、独立报道和社交内容必须分开。
- 跨届 Query 或数据源不一致时，不直接比较原始声量。

# 输出限制

报告必须写明数据来源、Query、时间、届次、地区、语言、去重、实体合并、排名指标、情绪口径和局限。
